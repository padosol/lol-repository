사용자가 전환하려는 대상 브랜치명: $ARGUMENTS

## 작업 순서

아래 단계를 순서대로 수행하세요.

### 1. 현재 브랜치 확인
`git branch --show-current` 명령어로 현재 브랜치명을 확인하고 기억합니다.

### 2. 대상 브랜치 유효성 검사
- `$ARGUMENTS`가 비어있으면 사용자에게 브랜치명을 물어보세요.
- 현재 브랜치와 대상 브랜치가 동일하면 "이미 해당 브랜치에 있습니다"라고 알려주고 중단합니다.

### 3. 브랜치 전환
`git checkout $ARGUMENTS` 명령어로 대상 브랜치로 전환합니다.
- 전환에 실패하면(브랜치가 존재하지 않는 등) 에러 메시지를 보여주고 중단합니다.
- 스테이징되지 않은 변경사항이 있어서 전환이 실패하면, 사용자에게 stash 할지 물어보세요.

### 4. 최신 코드 가져오기
`git pull` 명령어로 원격 저장소의 최신 코드를 가져옵니다.

### 5. 이전 브랜치 삭제 확인
사용자에게 이전 브랜치(1단계에서 기억한 브랜치)를 삭제할지 확인합니다.
- main, master, develop 같은 주요 브랜치인 경우 "주요 브랜치이므로 삭제하지 않겠습니다"라고 알려주고 삭제를 건너뜁니다.
- 사용자가 동의하면 `git branch -d` 명령어로 삭제합니다 (안전 삭제).
- merge되지 않은 커밋이 있어서 삭제가 실패하면, `git branch -D`(강제 삭제)를 할지 사용자에게 물어보세요.

### 6. 결과 보고
최종 상태를 간결하게 보고합니다:
- 전환된 브랜치
- pull 결과
- 이전 브랜치 삭제 여부
